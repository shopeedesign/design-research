# 设计竞品素材研究（design-research）

用于围绕某个 app / web 功能做竞品设计研究，重点不是泛分析，而是帮助设计师或产品同学快速找到可直接参考的界面截图、使用视频和页面级证据。

## 核心能力

- 根据产品名、模块或功能，自动推断最值得参考的竞品。
- 优先收集“能看到界面和操作过程”的素材，而不是泛资讯或讨论。
- 支持同时覆盖应用商店、视频/使用视角、其他截图来源三类证据。
- 对视频和截图做相关性过滤，避免返回大量低价值链接。
- 将结果按竞品聚合成表格，方便快速比较和继续深挖。

## 适用场景

- 想找某个功能的竞品界面，例如 `delivery notification`、`任务首页`、`岗位列表页`。
- 想看竞品的实际使用流程，而不是只看官方宣传文案。
- 需要为设计方向、页面结构或交互方案找真实参考素材。
- 用户提供产品名、功能和地区，希望快速得到一批可点开的证据。

## 不适用场景

- 需要完整商业分析、市场格局、SWOT 等纯策略型竞品分析。
- 只想收集用户评论、论坛争议或口碑讨论，而不关心界面与流程。
- 用户要实现设计稿自查或方案评审，此时更适合 `design-selfcheck`。
- 用户要理解 PRD 文档，而不是找设计参考，此时更适合 `prd-design-brief`。

## 输入与前置条件

- 建议输入：`当前产品名`、`目标模块/功能`、`可选地区`、`可选素材偏好`。
- 更细的功能描述会显著提高结果质量，但 skill 不会因为用户输入过宽而停止工作。
- 地区不是强制项，但建议尽量给出，避免结果过散。
- 不要求用户预先提供竞品名单，skill 会自行推断。

## 使用方式

### 推荐指令

```text
Use $design-research to find competitor UI for Shopee Express delivery notification in Southeast Asia.
```

```text
Use $design-research to find screenshots and usage videos for trunk logistics task homepages in China.
```

```text
Use $design-research to research competitor job-list homepages for third-party workers in Southeast Asia.
```

### 输入建议

如果用户愿意补充信息，优先补这两项：

- `更细的功能描述`
- `目标市场或地区`

例如：

- `J&T Express 派送功能界面`
- `delivery notification`
- `东南亚第三方工作人员岗位首页列表`

如果用户不补充，skill 仍然继续执行，并自动聚焦最可能相关的子流程。

## 工作流程

1. 识别用户真正要研究的产品类型与功能范围。
2. 推断一批最值得搜索的竞品，而不是依赖用户先给名字。
3. 按功能相关度筛选最值得深挖的竞品。
4. 搜索三类证据：
   - `应用商店相关`
   - `视频或者直接使用视角`
   - `其他相关截图等`
5. 对视频和截图做相关性过滤，剔除宣传内容和低价值结果。
6. 按竞品聚合成表格输出。

## 输出结果

结果固定按下列表头返回：

| Competitor | 应用商店相关 | 视频或者直接使用视角 | 其他相关截图等 |
|---|---|---|---|

输出规则：

- 一行对应一个竞品。
- 没有有效内容时直接留空。
- 每个单元格允许多条内容，但只保留高信号结果。
- `视频或者直接使用视角` 与 `其他相关截图等` 需要标记契合度。

常见标记方式：

- `high fit`：明确命中目标功能页面、状态或交互
- `medium fit`：有真实界面或流程，但未直接命中目标功能
- 不返回：纯宣传、无界面、无操作、低价值结果

## 目录说明

- `SKILL.md`：skill 的完整触发规则、流程和输出约束。
- `README.md`：当前这份统一格式的能力与用法说明。
- `agents/openai.yaml`：skill 的展示名称和默认 prompt 配置。

## 注意事项

- 这个 skill 的目标是“找得到可用设计证据”，不是输出一份泛竞品分析报告。
- 优先返回真实界面、真实流程、真实操作，不优先返回讨论帖或纯描述信息。
- 对视频和截图宁可少返回，也不要无脑堆链接。
- 当目标功能没有直接命中的视频或截图时，允许留空，不强行补低相关内容。
