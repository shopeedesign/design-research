# 竞品设计素材研究（design-research）

用于为 App 或网页产品的具体功能、页面、状态或流程寻找竞品 UI 参考素材。这个 skill 会主动推断相关竞品，搜索并核验截图、录屏、教程/使用视频、应用商店、帮助中心或页面级案例等界面素材，最后按竞品分组输出可判断价值的链接。

## 核心能力

- 基于产品类别、目标功能或具体流程，推断最相关的一组竞品。
- 搜索真实界面素材，而不是泛泛的市场信息或品牌介绍。
- 覆盖截图、屏幕录制、用户 walkthrough、教程视频、应用商店截图、帮助中心页面和页面级案例。
- 按 `high fit`、`medium fit` 等标签判断素材与目标功能的相关性。
- 在素材不足、无法访问或只能间接判断时，明确说明覆盖不足和待核验线索。

## 适用场景

- 设计师要为某个功能找竞品 UI 参考、截图或操作视频。
- 产品或交互方案改版前，需要了解同类产品如何处理页面结构、入口、状态反馈或异常流程。
- 设计评审需要真实竞品页面素材作为外部参考。
- 用户只知道产品方向或功能主题，希望 AI 先推断竞品再找素材。
- 需要同时覆盖中国市场和全球产品的界面素材来源。

## 不适用场景

- 泛泛分析市场规模、商业模式、增长策略、品牌定位或行业趋势。
- 只需要列竞品名单、公司名称或官网，不需要界面素材。
- 用户已经有截图，只想做 OCR、信息提取或截图整理。
- 用户要直接生成组件代码、页面代码或完整设计方案。

## 输入与前置条件

- 目标功能优先级最高，例如 `订单追踪页面`、`联系骑手交互`、`订阅付费墙`、`结账优惠码`。
- 产品或行业背景有助于推断竞品，例如外卖、电商、SaaS 协作工具、B 端导入流程。
- 市场或地区信息可提高素材相关性，例如中国市场、全球产品、移动端优先或 Web 优先。
- 偏好的素材类型可选，例如截图、录屏、教程视频、用户使用视频或应用商店截图。
- 如果当前环境不能打开网页、查看图片或查看视频内容，应输出搜索词、候选来源和待核验线索，不要假装已经核验。

## 使用方式

### 推荐指令

```text
帮我找一下外卖 App 里“配送失败 / 联系骑手”的竞品设计参考，最好有截图或视频。
```

```text
我们在做 SaaS 邀请团队成员流程，想看看 Notion、Slack、Linear、Figma 怎么处理邀请、权限和错误状态。
```

```text
找几个电商结账页优惠码输入和校验失败状态的真实 UI 素材，移动端优先。
```

### 建议同时提供的信息

- 当前产品：你正在设计或分析的产品名称。
- 目标功能：要研究的模块、页面、状态或交互流程。
- 目标市场：国内、海外、全球，或某个特定地区。
- 平台范围：iOS、Android、Web、桌面端或移动端优先。
- 素材偏好：截图、录屏、教程视频、用户 walkthrough、应用商店截图或都可以。

## 工作流程

1. 识别用户的产品类别、目标功能、平台范围和目标市场。
2. 当用户没有给出完整范围时，根据语言、产品名称、功能场景和平台线索做合理假设。
3. 推断 5-8 个相关竞品，并按功能相似度、交互参考价值和素材可获得性排序。
4. 将目标功能翻译成产品中更可能出现的术语，再组合中英文关键词搜索。
5. 优先检索界面素材库、产品截图来源、应用商店、视频平台、帮助中心和页面级案例。
6. 打开并检查候选链接，确认标题、页面内容、缩略图、截图、字幕或可见媒体能支持 fit 标签。
7. 按竞品分组输出表格，并补充覆盖不足或待核验线索。

## 输出结果

默认输出一个按竞品分组的表格：

| Competitor | 应用商店相关 | 视频或者直接使用视角 | 其他相关截图等 |
|---|---|---|---|

每条有效素材应尽量包含：

- 来源：例如 `App Store`、`Google Play`、`YouTube`、`B站`、`Mobbin`、`官方帮助中心`。
- fit 标签：`high fit` 或 `medium fit`。
- 可见素材：具体看到了哪个页面、状态、控件、入口或流程片段。
- 相关性说明：为什么对目标功能有参考价值。
- 局限性：平台、地区、时间、版本或只展示相邻流程等限制。

必要时会补充：

- `覆盖不足`：说明哪些竞品或素材模块没有找到强相关素材。
- `待核验线索`：放置可能相关但当前环境无法确认的候选来源或搜索关键词。

## 目录说明

- `SKILL.md`：skill 的触发条件、搜索流程、素材质量规则和输出规范。
- `README.md`：当前这份面向使用者的能力与用法说明。
- `agents/openai.yaml`：Codex UI 中展示 skill 的名称、简介和默认提示。
- `evals/evals.json`：用于评估 skill 输出质量的任务样例。
- `evals/trigger-evals.json`：用于评估 skill 触发边界的正反样例。

## 注意事项

- 不要把低价值链接塞满表格；宁可留空，也要保持素材高信号。
- 不要把纯品牌广告、无界面口播视频或泛产品页面当作有效素材。
- 只有可见内容支持时，才标注 `high fit` 或 `medium fit`。
- 如果素材无法打开、需要登录、地区受限或只能看到标题摘要，应放入 `待核验线索`，不要放入主表。
- 输出目标是“让设计师快速判断哪些链接值得打开”，因此每条素材都应说明看到什么、为什么相关。
