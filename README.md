# 竞品设计素材研究（design-research）

用于为 App、网页或 PC 端产品的具体功能、页面、状态或流程寻找竞品 UI 参考素材。这个 skill 会主动推断相关竞品，搜索并核验截图、录屏、教程/使用视频、应用市场/官方来源、帮助中心或页面级案例等界面素材，最后按竞品分组输出可判断价值的链接。

## 核心能力

- 基于产品类别、目标功能或具体流程，推断最相关的一组竞品。
- 当用户只给出泛类型功能而没有明确竞品时，先推断哪些产品或行业后台天然存在该流程，再形成竞品池。
- 搜索真实界面素材，而不是泛泛的市场信息或品牌介绍。
- 覆盖截图、屏幕录制、用户 walkthrough、教程视频、应用市场/官方截图、帮助中心页面和页面级案例。
- 按 `high fit`、`medium fit` 等标签判断素材与目标功能的相关性。
- 在素材不足、无法访问或只能间接判断时，明确说明覆盖不足和待核验线索。

## 适用场景

- 设计师要为某个功能找竞品 UI 参考、截图或操作视频。
- 产品或交互方案改版前，需要了解同类产品如何处理页面结构、入口、状态反馈或异常流程。
- 设计评审需要真实竞品页面素材作为外部参考。
- 用户只知道产品方向或功能主题，希望 AI 先推断竞品再找素材。
- 用户提出较泛的 PC/Web 功能，例如批量上传、批量导入、字段映射、日历视图、预约排期，需要先判断哪些软件存在类似交互。
- 需要同时覆盖中国市场和全球产品的界面素材来源。

## 不适用场景

- 泛泛分析市场规模、商业模式、增长策略、品牌定位或行业趋势。
- 只需要列竞品名单、公司名称或官网，不需要界面素材。
- 用户已经有截图，只想做 OCR、信息提取或截图整理。
- 用户要直接生成组件代码、页面代码或完整设计方案。

## 输入与前置条件

- 目标功能优先级最高，例如 `订单追踪页面`、`联系骑手交互`、`订阅付费墙`、`结账优惠码`。
- 产品或行业背景有助于推断竞品，例如外卖、电商、SaaS 协作工具、B 端导入流程。
- 市场或地区信息可提高素材相关性，例如中国市场、全球产品、移动端优先或 Web 优先。
- 偏好的素材类型可选，例如截图、录屏、教程视频、用户使用视频或应用市场截图。
- 如果当前环境不能打开网页、查看图片或查看视频内容，应输出搜索词、候选来源和待核验线索，不要假装已经核验。

## 使用方式

### 推荐指令

```text
帮我找一下外卖 App 里“配送失败 / 联系骑手”的竞品设计参考，最好有截图或视频。
```

```text
我们在做 SaaS 邀请团队成员流程，想看看 Notion、Slack、Linear、Figma 怎么处理邀请、权限和错误状态。
```

```text
找几个电商结账页优惠码输入和校验失败状态的真实 UI 素材，移动端优先。
```

```text
搜索 PC 端关于批量上传的交互界面资源。
```

```text
输出 PC 端日历相关的界面资源。
```

### 建议同时提供的信息

- 当前产品：你正在设计或分析的产品名称。
- 目标功能：要研究的模块、页面、状态或交互流程。
- 目标市场：国内、海外、全球，或某个特定地区。
- 平台范围：iOS、Android、Web、桌面端或移动端优先。
- 素材偏好：截图、录屏、教程视频、用户 walkthrough、应用市场/官方截图或都可以。

## 工作流程

1. 识别用户的产品类别、目标功能、平台范围和目标市场。
2. 当用户没有给出完整范围时，根据语言、产品名称、功能场景和平台线索做合理假设。
3. 如果用户没有明确竞品，先做“功能存在场景推断”，列出最可能拥有该流程的软件类型，并从中选择具体产品。
4. 推断 5-8 个相关竞品，并按功能相似度、交互参考价值和素材可获得性排序。
5. 先按资源类型强制执行完整固定搜索词组，再用目标功能、页面词、状态词或平台词做二轮精筛。
6. 优先检索界面素材库、产品截图来源、应用市场/官方来源、视频平台、帮助中心和页面级案例。
7. 打开并检查候选链接，确认标题、页面内容、缩略图、截图、字幕或可见媒体能支持 fit 标签。
8. 按竞品分组输出表格，并补充覆盖不足或待核验线索。

## 固定搜索词组

第一轮搜索是硬约束：对每个竞品、每个资源平台，都要把对应平台类型下的所有搜索词逐条执行一遍；不能组合搜索词，也不能因为前一个搜索词已有结果就跳过后面的搜索词。用户给出的精确目标词只用于下钻核验，不进入第一轮搜索词。

视频平台固定搜索词：

```text
[竞品名称] APP
[竞品名称] UI
[竞品名称] Page
[竞品名称] 使用教程
怎么使用[竞品名称]
```

非视频平台固定搜索词：

```text
[竞品名称]
[竞品名称] APP
[竞品名称] UI
[竞品名称] Page
```

## 泛类型需求处理方式

当用户的需求不是某个明确竞品，而是类似 `PC 端批量上传`、`PC 端日历界面`、`后台批量导入` 这类泛类型功能时，skill 会先把功能归因到常见软件场景，再开始检索。

示例：

- `PC 端批量上传`：优先推断电商商家后台、CRM/营销工具、协作表格、项目管理、网盘/素材管理等竞品池。
- `PC 端日历界面`：优先推断 Google Calendar、Outlook Calendar、Notion Calendar、飞书/Lark Calendar、Calendly、Airtable、ClickUp 等竞品池。
- `CSV/Excel 导入字段映射`：优先推断 HubSpot、Salesforce、Mailchimp、Airtable、monday.com、Shopify 等竞品池。

如果循环搜索多轮后仍没有足够可核验素材，skill 会说明当前覆盖不足，并建议缩窄行业、数据类型、平台、用户角色或流程阶段。

## 输出结果

默认输出一个按竞品分组的表格：

| Competitor | 应用市场/官方来源 | 视频或者直接使用视角 | 其他相关截图等 |
|---|---|---|---|

每条有效素材应尽量包含：

- 来源：例如 `应用宝`、`华为应用市场`、`Google Play`、`YouTube`、`B站`、`Mobbin`、`UI Notes`、`Page Flows`、`官方帮助中心`。
- fit 标签：`high fit` 或 `medium fit`。
- 可见素材：具体看到了哪个页面、状态、控件、入口或流程片段。
- 相关性说明：为什么对目标功能有参考价值。
- 局限性：平台、地区、时间、版本或只展示相邻流程等限制。

必要时会补充：

- `覆盖不足`：说明哪些竞品或素材模块没有找到强相关素材。
- `待核验线索`：放置可能相关但当前环境无法确认的候选来源或搜索关键词。

## 目录说明

- `SKILL.md`：skill 的触发条件、搜索流程、素材质量规则和输出规范。
- `README.md`：当前这份面向使用者的能力与用法说明。
- `agents/openai.yaml`：Codex UI 中展示 skill 的名称、简介和默认提示。
- `evals/evals.json`：用于评估 skill 输出质量的任务样例。
- `evals/trigger-evals.json`：用于评估 skill 触发边界的正反样例。

## 注意事项

- 不要把低价值链接塞满表格；宁可留空，也要保持素材高信号。
- 不要把纯品牌广告、无界面口播视频或泛产品页面当作有效素材。
- 只有可见内容支持时，才标注 `high fit` 或 `medium fit`。
- 如果素材无法打开、需要登录、地区受限或只能看到标题摘要，应放入 `待核验线索`，不要放入主表。
- 输出目标是“让设计师快速判断哪些链接值得打开”，因此每条素材都应说明看到什么、为什么相关。
