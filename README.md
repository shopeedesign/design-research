# design-research

用于为 App、网页或 PC 端产品的具体功能、页面、状态或流程寻找竞品 UI 参考素材。它会先推断竞品池，再按产品和渠道检索真实界面素材，核验截图、录屏、应用市场截图、截图/流程库、画板入口、帮助中心、教程视频和页面级案例。

## 能力范围

- 根据产品类别、功能、页面、状态或流程推断 5-8 个直接竞品，并补充少量相邻产品。
- 按竞品分别搜索，不用泛词反复凑结果。
- 覆盖应用商店、UI 素材库、截图/流程库、视频平台、官方资料和帮助中心。
- 对花瓣只返回站内搜索入口，不使用 `site:huaban.com` 外部搜索结果代替站内搜索。
- 在主表外增加 `Google 图片补充`，通过 Google Images 初筛含 UI 的图片结果，再下钻来源页二次判断。
- 输出覆盖不足和待核验线索，避免把无法核验的内容写成确定结论。

## 输出结构

默认直接在聊天中输出：

1. 研究范围
2. 竞品选择逻辑
3. 主结果表
4. Google 图片补充
5. 非直接竞品/主题合集补充
6. 覆盖不足
7. 待核验线索

主结果表固定为四列：

| 产品 | 应用商店 | 界面/截图 | 其他来源 |
|---|---|---|---|

每条链接使用：

```markdown
[来源名称](https://example.com)｜素材类型｜核验状态/fit｜简短备注
```

## Google 图片补充

主结果表后必须输出 `Google 图片补充`。检索逻辑是：

1. 将用户关键词、竞品名、页面/功能词和平台词放到 Google Images / 图片搜索中检索。
2. 先看图片结果是否含真实 UI 信息，例如手机屏幕、网页页面、流程截图、弹窗、设置页、表单、列表或错误态。
3. 对疑似有效结果打开来源页二次判断。
4. 只返回来源页链接，不返回图片直链、Google Images 搜索页、`imgurl`、`gstatic`、`encrypted-tbn` 或 CDN 图片文件。
5. 合格结果输出 5-8 条；不足 5 条时只输出已核验的结果。如果都不符合，输出 `Google 图片补充：没有符合结果。`

## 花瓣规则

花瓣是低自由度渠道，只返回站内搜索入口：

- App：`[产品名称] app`
- PC/Web：`[产品名称] UI`

最终回答不要输出独立的“花瓣检索记录”或“集合平台检索记录”板块。花瓣入口如果保留，只放入主结果表的 `界面/截图` 列或 `待核验线索`。

## 目录说明

- `SKILL.md`：skill 触发条件、核心流程和核验口径。
- `references/output-format.md`：最终回答结构、四列表、fit 标签、Google 图片补充和自检规则。
- `references/platform-search.md`：渠道优先级、花瓣入口、Pinterest 规则、Google Images 补充检索和各渠道兜底。
- `scripts/collection-query-plan.mjs`：生成花瓣站内搜索入口。
- `scripts/external-link-helper.mjs`、`scripts/launch-external-link-helper.mjs`：生成本地 HTML/聚合页时的外链辅助脚本。
- `agents/openai.yaml`：Codex UI 中展示 skill 的名称、简介和默认提示。
- `evals/`：skill 输出质量和触发边界样例。

## 注意事项

- 主结果只放 `high fit` 或 `medium fit` 的强链接。
- 不返回纯宣传、无可见 UI、无真实使用过程或无法核验的链接。
- 视频未完整播放时标注 `画面未完整核验`、`缩略图可见` 或 `仅标题/摘要支撑`。
- Pinterest 默认不作为 P0 渠道，除非用户明确要求、提供链接，或其他渠道明显不足且需要灵感补充。
- 默认不要返回 App Store，除非用户明确需要 iOS / iPhone / iPad / App Store 界面素材。
