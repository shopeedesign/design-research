# Platform Search Reference

用于选择检索渠道、生成花瓣站内搜索入口，以及在写“未找到/覆盖不足”前完成兜底。

## 渠道优先级

P0 要同时覆盖三类素材：截图/流程库、应用市场或官方截图、视频/直接使用视角。P1 用于补充高价值但不稳定的来源。P2 只用于冷门产品、地区性产品或用户明确要求。

| 市场 | P0 | P1 | P2 |
|---|---|---|---|
| 中国大陆 | UI Notes、花瓣画板、B站、抖音、小米应用商店、应用宝、官方帮助中心/官网 | 小红书、公众号、华为/OPPO/vivo/酷安/豌豆荚 | APKPure、APKCombo、360、百度手机助手、知乎、微博、论坛搬运 |
| 全球/欧美 | Mobbin、Google Play、Page Flows、YouTube、官网/帮助中心、App Store 仅在用户明确要 iOS 时使用 | UI Sources、App Fuel、Design Vault、Refero、TikTok | Pinterest、Reddit、Product Hunt、Medium、Behance、Dribbble、论坛搬运 |
| 日本 | Google Play JP、YouTube 日文教程、Appliv、官网/帮助中心、App Store JP 仅在用户明确要 iOS 时使用 | note、日文博客、TikTok | Pinterest、X、个人博客、APK 镜像 |
| 韩国 | Google Play KR、YouTube 韩文视频、ONE Store、Naver Blog、官网/帮助中心、App Store KR 仅在用户明确需要 iOS 时使用 | Naver Cafe、TikTok、Tistory | Pinterest、APK 镜像、论坛搬运 |
| 东南亚/亚洲 | Chamjo、Mobbin、UI Sources、Google Play 本地区、YouTube、TikTok、官方帮助中心 | UIguana、Page Flows、APKPure、APKCombo、本地媒体 | Pinterest、本地论坛、Facebook 群、博客搬运 |

Pinterest 不作为 P0，也没有必搜闸门；只有用户明确要求、提供链接，或 P0/P1 覆盖明显不足且需要灵感补充时才使用。

## 渠道归类

- 截图/流程库：UI Notes、Mobbin、Chamjo、Page Flows、UI Sources、App Fuel、Design Vault、ScreensDesign、UIguana。
- 应用市场：Google Play、App Store、中国安卓市场、ONE Store。
- 画板/合集：花瓣；Pinterest 仅作可选补充。
- 图片搜索补充：Google Images / Google 图片搜索，用于发现主渠道漏掉的 UI 来源页，不替代 P0/P1。
- 视频/直接使用：B站、YouTube、抖音、TikTok。
- 官方资料：官网、帮助中心、更新日志、开发者博客、公众号官方文章。
- 社区内容：小红书、知乎、Naver Blog、Reddit、Medium、本地论坛；公开搜索限制大，除非用户要求，不做绝对否定。

## 花瓣画板入口规则

花瓣采用站内搜索入口返回策略。当前环境无法可靠读取花瓣站内结果列表，因此不要使用外部搜索引擎结果代替站内结果，也不要筛选或排序具体画板。

优先运行脚本生成完整清单：

```bash
node scripts/collection-query-plan.mjs --market china --surface app --products "产品A,产品B"
node scripts/collection-query-plan.mjs --market china --surface pc --products "产品A,产品B"
```

参数：

- `--market`：`china`、`cn`、`both`、`unknown` 生成花瓣入口；`global/overseas` 默认不生成花瓣入口，除非用户明确需要国内素材。
- `--surface`：`app` 或 `pc`。`app` 使用 `[产品名] app`，`pc` 使用 `[产品名] UI`。
- `--products`：竞品和别名，用逗号分隔。

如果脚本不可用，按下面规则手工生成同等清单。

### 固定查询词和 URL

对每个产品或别名只生成一个花瓣站内搜索入口：

- App：`[产品名] app`
- PC 端/网页端：`[产品名] UI`

站内 URL 模板：

- `https://huaban.com/search?type=board&q=[URL编码关键词]&search_word_type=手动输入搜索词&original=[URL编码关键词]`

### 禁止的花瓣替代策略

- 不使用 `site:huaban.com/boards ...` 作为花瓣画板结果。
- 不使用 `site:huaban.com/pins ...` 作为花瓣画板结果。
- 不用外部搜索摘要推断花瓣站内前三个画板。
- 不判断花瓣站内搜索入口里是否有结果、结果数量或排序。

输出状态统一写 `站内搜索入口` 和 `需浏览器核验`。如果用户提供浏览器截图或可见结果，再按用户提供信息记录。

### 用户提供花瓣链接

用户直接提供花瓣画板、pin、topic-board 或截图时，必须保留该链接，但只能记录已知信息：

- 能打开并看到内容时，按正常核验进入主表、补充区或待核验。
- 自动打不开时，状态写 `用户提供 / 需浏览器核验` 或 `自动抓取受限`。
- 不补充未看见的 pins、采集数、画板标题或排序。

## Pinterest 规则

Pinterest 不作为 P0，不触发硬闸门。默认不搜索、不返回 Pinterest。

只有以下情况才使用：

- 用户明确要求 Pinterest。
- 用户提供 Pinterest pin/board 链接。
- P0/P1 渠道覆盖不足，且需要灵感补充；此时必须说明 Pinterest 不是核心竞品证据。

处理规则：

- 可打开具体 pin/board 时，按实际可见内容标注。
- 搜索页或 board 搜索入口不可靠，不要宣称前三个 board 或结果排序。
- 不进入主表，除非用户明确要求 Pinterest 或该链接能直接展示真实竞品 UI。

### 内部记录

花瓣入口触发时可在内部记录哪些入口已经生成，但最终回答不要输出独立的“花瓣检索记录”或“集合平台检索记录”板块。脚本生成的 `Evidence table` 也只作为内部辅助，不复制到最终答复。

花瓣入口状态只能使用：`站内搜索入口`、`需浏览器核验`、`用户/浏览器可见`、`自动抓取受限`、`已打开核验`。如果只返回入口，不要写 `无可核验结果`。需要展示花瓣入口时，只放在主结果表的 `界面/截图` 列或 `待核验线索`。

## Google Images 补充检索

Google Images 用于最终回答中的补充列表，目标是通过图片结果发现含真实 UI 的来源页。它是补充渠道，不替代 UI 素材库、应用市场、视频或官方资料。

### 查询词

优先用用户原始关键词；再按产品、页面和平台扩展：

- `[用户关键词] app UI`
- `[用户关键词] app 截图`
- `[产品名] [页面/功能词] app UI`
- `[产品名] [页面/功能词] 截图`
- `[产品名] mobile app screenshots`
- PC/网页端使用 `[产品名] [页面/功能词] web UI` 或 `[产品名] dashboard UI`

### 初筛图片结果

只把图片结果作为初筛信号。保留能看出真实 UI 信息的结果，例如手机屏幕、网页页面、后台界面、流程截图、表单、列表、卡片、弹窗、设置页、错误态或操作路径。

直接排除：

- 只有 Logo、品牌海报、宣传 KV、人物照片、插画、纯图标或营销横幅。
- 缩略图看不清 UI，或无法判断是不是目标产品/目标功能。
- 与用户目标明显无关的模板、概念稿、广告图或泛行业素材。

### 下钻二次判断

对初筛通过的图片结果，必须打开来源页二次判断。最终只返回来源页链接，不返回图片直链、Google Images 搜索页、`imgurl`、`gstatic`、`encrypted-tbn`、CDN 图片文件或只打开单张图片的页面。

来源页可包括文章、视频、截图合集、教程页、应用市场页、设计案例页或含真实截图的网页。进入补充列表前至少确认：

- 来源页里仍能看到对应 UI，或页面文字/视频/截图明确支撑该 UI。
- 链接不是图片文件，也不是搜索结果页。
- 该结果对目标页面、流程、状态或竞品有 high fit / medium fit 的参考价值。
- 旧版本、地区差异、视频未完整播放、来源页截图不完整等局限已经标注。

### 输出取舍

Google 图片补充列表返回 5-8 条来源页链接；不足 5 条时只返回已核验的合格链接，不用低价值内容凑数。优先级：

1. 图片结果和来源页都清晰展示目标 UI。
2. 来源可信、可访问、无需登录。
3. 覆盖不同竞品、不同页面或不同流程。
4. 时间更近、平台更贴近用户目标。

如果下钻后都不符合，最终输出 `Google 图片补充：没有符合结果。`

## 其他渠道规则

### UI 素材库和截图库

除固定查询组外，至少补查 `site:[平台域名] [竞品名称]`、`site:[平台域名] [竞品名称] app`、产品别名、英文名、包名或开发者名。命中单图时，继续找同源 app、flow、collection 或 profile 集合页。

### 应用市场

中国大陆 Android 优先返回小米应用商店和应用宝；两者都有可核验截图时，不再补同质镜像市场。只有缺失、无截图、不可访问或用户明确要求更多市场时，才补华为、OPPO/vivo、酷安、豌豆荚、360、APK 聚合页等。

搜索除应用名外还要补包名、开发者名、旧名/新名、中文名/英文名。某个市场无截图，不能推断所有市场无截图。

默认不要返回 App Store；只有用户明确提出 iOS、iPhone、iPad、App Store 或苹果端界面时才搜索和返回。

### 视频平台

视频渠道用于补操作路径、动效、提交反馈和异常状态，属于 P0。除视频固定查询组外，补查 `site:[平台域名] [竞品名称]`、`[竞品名称] walkthrough`、`[竞品名称] tutorial`、`[竞品名称] 使用/教程/测评`、产品别名和目标页面/功能词。

可依据画面、缩略图、章节、字幕、标题或摘要判断相关性；未完整播放时要标注 `画面未完整核验`、`缩略图可见` 或 `仅标题/摘要支撑`。

### 官方资料

已知官方域名时，补查 `site:[官方域名] [目标功能/页面词]`、`site:[支持域名] [竞品名称]`、产品内术语、旧名/新名和功能同义词。官方域名未知时写 `未确认官方域名`，不要写成官方没有素材。

## 否定结论

在声明某平台未找到、无可核验结果、只有安装页或未确认有操作视频前，必须先判断是否存在站内搜索弱、搜索引擎索引不全、别名遗漏、页面分散、登录或地区限制。存在这些风险时，必须按该平台规则完成兜底，并覆盖已知别名、旧名/新名、中文名/英文名、包名、开发者名或地区名。
