# Platform Search Reference

用于按目标市场选择目标平台、处理特殊平台限制，并在写“未找到/覆盖不足”前完成必要兜底。

本文件只规定“去哪里搜、怎么形成候选素材池、特殊平台怎么处理”。最终结果怎么呈现，以 `references/output-format.md` 为准。

## 核心原则

- 目标平台指所有可用于发现 UI 资源的渠道，包括应用商店、截图/流程库、视频平台、Google Images、Google Videos、官方资料、画板/board、社区内容等。
- 按目标市场选择目标平台后，对每个竞品在目标平台中执行检索、下钻核验和筛选。
- 产品类型/市场检索中，只要某产品进入直接竞品主结果表，就按直接竞品最低检索深度执行，不能只返回最容易找到的应用商店单链。
- 选定竞品后，先为每个直接竞品寻找 1 个且仅 1 个非 iOS 应用商店截图来源，作为基础截图证据；应用商店结果不能替代后续必跑渠道。
- P0 / P1 / P2 是目标平台优先级，不是最终输出分类。P0 中标注为“必跑”的渠道必须执行；其他 P0/P1/P2 渠道根据必跑渠道召回、目标页面覆盖和核验需要再选择。
- Google Images / Google Videos 是目标平台发现入口。命中结果必须下钻到来源页判断，最终只返回来源页，不返回 Google 搜索页、图片/视频直链、缩略图链接或 CDN 资源。
- 花瓣等特殊平台按本文件的限制执行。
- 禁止调用、搜索或返回 iOS App Store / Apple App Store。即使用户明确提出 iOS、iPhone、iPad 或 App Store，也不要使用 App Store；改用非 iOS 应用商店、截图库、视频、官网帮助或 Google Images / Google Videos 下钻来源。

## 目标平台优先级

P0 默认优先覆盖截图/流程库、应用市场或官方截图、视频/直接使用视角等核心素材来源。表格中的“必跑”渠道必须检索；未标必跑的渠道根据召回质量和覆盖缺口选择。

| 全球/欧美 | 必跑：Mobbin、Page Flows、YouTube、Google Images、Google Videos；基线：1 个非 iOS 应用商店截图；可选：TikTok、官网/帮助中心 | UI Sources、App Fuel、Design Vault、Refero | Pinterest、Reddit、Product Hunt、Medium、Behance、Dribbble、论坛搬运 |

## 目标平台归类

- 截图/流程库：UI Notes、Mobbin、Chamjo、Page Flows、UI Sources、App Fuel、Design Vault、ScreensDesign、UIguana。
- 应用市场：Google Play、中国安卓市场、ONE Store、APKPure、APKCombo、AppBrain 等非 iOS 来源；不包含 iOS App Store / Apple App Store。
- 视频/直接使用：B站、YouTube、抖音、TikTok、Google Videos。
- Google Images / Google Videos：作为发现入口使用。通过它们发现的截图、视频或页面，必须打开真实来源页核验；最终按真实来源类型归类，例如 AppBrain 截图页放入 `界面/截图`，YouTube 视频放入 `界面/截图`，应用商店页放入 `应用商店`。不要返回 Google 搜索结果页，也不要把 Google Images / Google Videos 写成最终来源。
- 官方资料：官网、帮助中心、更新日志、开发者博客、官方公众号文章。
- 画板/合集：花瓣、Pinterest 等。
- 社区内容：小红书、知乎、Naver Blog、Reddit、Medium、本地论坛；公开搜索限制大，除非用户要求，不做绝对否定。

## 固定查询词与返回数量

对每个竞品，在目标平台中依次检索以下固定查询词：

- `[竞品名称]`
- `[竞品名称] APP`
- `[竞品名称] UI`
- `[竞品名称] Page`
- `[竞品名称] 使用教程`
- `怎么使用[竞品名称]`

执行要求：

1. 所有查询词都要检索完，不因前几个查询已有结果就停止。
2. 检索得到的是候选素材池，不代表最终可返回。
3. 同一竞品有别名、旧名/新名、中文名/英文名、包名、开发者名或地区版本时，要用这些名称变体补充检索。
4. 视频平台要在固定查询词之外追加视频专项词，具体按“视频平台规则”执行。
5. 如果用户指定功能、页面或状态，优先在候选素材池的下钻核验阶段判断是否覆盖该目标；不要默认把功能词、页面词或状态词拼进每个固定查询词。
6. 只有固定查询组召回不足，或目标词本身是强检索词时，才追加目标关键词做补充检索。强检索词包括产品内专有功能名、行业固定术语、平台官方命名或用户明确指定的独特入口名。
7. 花瓣等特殊平台的查询词受平台规则限制时，按对应特殊平台规则执行。

返回数量控制：

1. 所有查询词检索完成后，对同一竞品的候选素材统一下钻核验、去重、筛选和排序。
2. 每个直接竞品最终最多返回 6 条高价值资源；合格结果不足 6 条时，只返回已核验的 high fit / medium fit 结果，不用低价值链接凑数。
3. 应用商店 必须且只能返回 1 条非 iOS 来源，选择截图数量、截图质量、版本/更新时间、目标市场匹配度和访问稳定性综合最高的应用商店链接。
4. 界面/截图 优先保留截图/流程库、视频/录屏、Google Images / Google Videos 下钻后的真实来源页等能看到真实 UI 的资源。
5. 其他来源 保留官网、帮助中心、教程文章、社区资料等能补充页面、流程、状态或使用方式的辅助资源。
6. 相同来源页通过多个查询词命中时，只保留一次；内容高度重复的资源，保留信息量更高、可见 UI 更多或来源更可靠的一条。

最低覆盖要求：

1. 每个进入主结果表的直接竞品，至少尝试覆盖 `应用商店`、`界面/截图`、`视频/官方/教程` 中的两类资源；其中应用商店只能是 1 条非 iOS 来源。
2. 如果应用商店之外存在 high fit / medium fit 来源，不能只返回应用商店链接。
3. 如果最终只能返回应用商店单链，必须先完成目标市场必跑渠道、必要兜底渠道和名称变体兜底，并在 `覆盖不足` 中说明没有找到其他可核验来源。
4. 产品类型/市场检索下，直接竞品的最低覆盖要求优先于补充参考；不要为了输出补充参考而牺牲主竞品检索深度。

## 下钻核验标准

打开候选页面、视频页、截图集合、帮助文档或应用市场截图后，再判断是否可返回。不要把搜索结果标题当事实。

下钻时至少确认：

- 素材类型：应用商店截图、截图合集、流程库、视频、帮助文档、官网资料、教程文章、画板/board 等。
- 可见内容：是否能看到真实 UI、页面、流程、状态、控件、入口或交互动作。
- 目标匹配：是否覆盖用户指定的竞品、产品类型、市场、功能、页面或状态。
- 参考价值：界面数量是否足够、信息是否清晰、是否有连续流程、是否比同类结果更有价值。
- 局限性：旧版本、地区差异、不同平台、登录限制、视频未完整核验、只展示相邻流程、只能摘要支撑等。
- 元信息：版本、地区、平台、发布时间或来源可信度，能确认时尽量记录。

## Google Images / Google Videos 规则

Google Images / Google Videos 是目标平台发现入口，不是最终来源。Google Videos 指 Google 搜索结果中的 Videos 标签页，用于发现 YouTube、官网、社媒或其他视频来源页。Google Images 指 Google 搜索结果中的 Images 标签页，用于发现包含真实 UI 截图的来源页。

检索方式：

- 使用固定查询词逐个检索；如有别名、旧名/新名、中文名/英文名、包名、开发者名或地区版本，按名称变体补充检索。
- 用户原始关键词只用于理解目标和下钻判断；不要默认把其中的功能词、页面词或状态词拼进每个固定查询词。
- 只有固定查询组召回不足，或目标词本身是强检索词时，才追加目标关键词做补充检索。


保留条件：

- 图片或视频结果能看出真实 UI 信息，或来源页明确包含真实界面素材。
- 来源页打开后二次确认，仍能支撑对应 UI 判断。
- 来源页能提供截图、视频、教程、应用市场页、设计案例页、帮助文档或含真实截图的网页。

排除条件：

- 只有 Logo、品牌海报、宣传 KV、人物照片、插画、纯图标或营销横幅。
- 缩略图看不清 UI，或无法判断是不是目标产品/目标功能。
- 来源页打不开、来源页无 UI、来源页与搜索结果不一致。
- 链接是 Google 搜索页、图片直链、视频直链、`imgurl`、`gstatic`、`encrypted-tbn`、CDN 文件、缩略图链接或只打开单张图片的页面。

最终只返回来源页链接，并按来源页类型归类到 `应用商店`、`界面/截图` 或 `其他来源`，不要把 Google Images / Google Videos 当作最终来源名称。

## 应用市场规则

搜索除应用名外，还要覆盖别名、旧名/新名、中文名/英文名、包名、开发者名和地区版本。某个市场无截图，不能推断所有市场无截图。

应用商店截图要单独去重。同一竞品命中多个应用商店时，不要默认返回多个应用商店链接。先比较：

- 截图数量。
- 截图清晰度和信息量。
- 版本/更新时间。
- 目标市场匹配度。
- 访问稳定性。
- 是否覆盖不同平台版本。

默认只返回综合价值最高的一个非 iOS 应用商店链接。即使不同应用商店截图明显不同、覆盖不同平台版本，或用户明确需要多应用商店对比，也不要在主结果表中返回多个应用商店链接；多出的可作为待核验线索或不返回。

禁止调用、搜索或返回 iOS App Store / Apple App Store。用户要求苹果端时，说明当前规则不使用 App Store，并通过截图库、视频、官网帮助、Google Images / Google Videos 下钻来源补充苹果端或相近界面证据。

中国大陆 Android 可优先比较小米应用商店、应用宝、华为/OPPO/vivo/酷安/豌豆荚等；海外和东南亚可优先比较 Google Play、APKPure、APKCombo 等。

## 截图/流程库规则

截图/流程库用于寻找真实页面、流程、状态和组件截图。优先选择能展示连续流程、多个页面或明确页面状态的结果。

命中单张截图时，继续查看同源 app、flow、collection、profile 或相关集合页，优先返回信息量更高的集合页。

### UI Notes 规则

UI Notes 是中国大陆必跑截图渠道。检索时不能只停在单张 `/pin/` 截图页，必须额外寻找产品级 `/app/` 集合页。

执行方式：

- 对每个竞品执行固定查询词后，追加 `site:uinotes.com/app [竞品名称]`、`[竞品名称] UI Notes app`、`[竞品名称] UI Notes [截图]` 兜底。
- 如果先命中 `uinotes.com/pin/...` 单张截图页，必须继续打开同源产品信息，或用上面的 `/app/` 查询找到 App 集合页。
- 同一产品同时存在 `/app/` 集合页和 `/pin/` 单张截图页时，默认返回 `/app/` 集合页；只有集合页无法打开、无法确认产品，或单张截图精准覆盖用户指定页面/状态且集合页不支持该判断时，才返回 `/pin/`。
- 打开 `/app/` 集合页后，记录产品名、版本、截图数量、功能分类、组件分类和登录限制。即使“登录后浏览全部截图”，只要页面公开显示截图数量和功能分类，仍可作为高价值截图合集返回，并在备注中说明部分截图需登录。
- 如果 `/pin/` 页面在自动环境正文为空或抓取异常，不要直接判定 UI Notes 无可用结果；改用 `/app/` 查询和 Google Images 下钻兜底。

如果同一竞品在多个截图/流程库都有结果，优先保留：

- 覆盖目标功能/页面/状态更明确的结果。
- 页面数量更多或流程更连续的结果。
- 版本、平台、地区更贴近用户目标的结果。
- 访问稳定、无需登录、可复核的结果。

## 视频平台规则

视频平台是重要目标平台，不能仅作为可选补充。它用于发现真实操作过程、页面流转、动效、提交反馈和异常状态，尤其适合补足静态截图无法覆盖的流程细节。

检索时使用固定查询词，并结合产品别名、旧名/新名、地区版本、包名或开发者名。固定查询组召回不足时，可补充 `walkthrough`、`tutorial`、`使用`、`教程`、`测评` 等强相关词。

### B站 / 录屏类视频补充规则

B站搜索结果里常混入宣传片、快递员日常、打包、测评和口播；这些标题相关但 UI 价值低。相反，高价值 UI 录屏常用“软件”“如何用”“怎么选择”等生活教程标题，不一定包含 `APP`、`UI` 或“下单”。因此国内市场跑 B站时，除固定查询词外，每个竞品必须追加以下视频专项检索：

- 通用录屏词：`[竞品名称] 软件`、`[竞品名称] 录屏`、`[竞品名称] 屏幕录制`、`如何用[竞品名称]`、`怎么用[竞品名称]`、`[竞品名称] 怎么选择`。
- 流程动词词：按用户目标和产品领域追加真实动作词，例如 `下单`、`寄件`、`查件`、`退货退款`、`支付`、`绑定`、`预约`、`取消`、`认证`、`申请`、`发票`。
- 物流/快递类特定词：`[竞品名称] 服务点自寄`、`[竞品名称] 上门取件`、`[竞品名称] 自寄`、`[竞品名称] 特快`、`[竞品名称] 寄快递`、`[竞品名称] 查快递`、`[竞品名称] 取件码`。

发现一个 high fit 或疑似 high fit 的录屏视频后，必须继续追踪：

- 同合集 / `ugc_season` 中的其他视频。
- 同 UP 主最近发布的同类“如何用软件 / Screenrecorder / 录屏”视频。
- 视频页相关视频、推荐视频、同标题模板的视频。

优先保留满足以下条件的视频：

- 标题、简介或分 P 信息包含“软件”“如何用”“怎么选择”“上门取件”“服务点自寄”等真实任务词。
- `part`、文件名或元信息出现 `Screenrecorder`、`screen record`、`录屏` 等信号。
- 首帧、缩略图或画面能看到手机桌面、App 图标、首页、表单、弹窗、步骤页、支付页、地图选择、服务点选择或提交结果。
- 即使播放量不高，只要能看到真实 UI 或连续流程，也优先于播放量更高但无界面的宣传、口播或线下日常视频。

如果 B站搜索接口触发风控或召回不稳定，不要直接判定“未找到”；改用站外 `site:bilibili.com/video` 检索结合上述视频专项词，并对命中的 BV 号调用视频详情页或可访问页面核验。最终返回 B站视频页，不返回搜索页。

可依据画面、缩略图、章节、字幕、标题或摘要判断相关性；未完整播放时必须标注 `画面未完整核验`、`缩略图可见` 或 `仅标题/摘要支撑`。

纯口播、无屏幕内容、只展示宣传片或无法判断真实界面的结果不要进入主结果。

## 官方资料规则

官方资料包括官网、帮助中心、更新日志、开发者博客和官方文章。

已知官方域名或支持域名时，围绕竞品名称、产品内术语、功能词、页面词和状态词检索。官方资料如果包含真实 UI 截图或操作路径，可以进入 `界面/截图` 或 `其他来源`；如果只有规则说明没有 UI，只能作为辅助资料。

官方域名未知时写 `未确认官方域名`，不要写成官方没有素材。

## 花瓣画板入口规则

花瓣是低自由度渠道。当前环境无法可靠读取花瓣站内搜索结果或画板内容，因此不要筛选、排序或宣称找到具体画板；只返回可点击的花瓣站内搜索入口。

触发范围：

- 中国大陆市场可生成花瓣入口。
- `global` / `overseas` / 海外市场默认不生成花瓣入口，除非用户明确需要国内素材、画板素材或花瓣链接。
- 用户直接提供花瓣链接时，必须保留该链接并按可见信息记录。

搜索词规则：

- App：`[产品名称] app`
- PC 端/网页端：`[产品名称] UI`

优先运行脚本生成完整清单：

```bash
node scripts/collection-query-plan.mjs --market china --surface app --products "产品A,产品B"
node scripts/collection-query-plan.mjs --market china --surface pc --products "产品A,产品B"
```

如果脚本不可用，按固定 URL 手工生成站内搜索入口：

```text
https://huaban.com/search?type=board&q=[URL编码关键词]&search_word_type=手动输入搜索词&original=[URL编码关键词]
```

禁止策略：

- 不使用 `site:huaban.com/boards` 作为花瓣画板结果。
- 不使用 `site:huaban.com/pins` 作为花瓣画板结果。
- 不用外部搜索摘要推断花瓣站内前三个画板。
- 不判断花瓣站内搜索入口里是否有结果、结果数量或排序。
- 不补充未看见的 pins、采集数、画板标题或排序。

状态写法：

- 只返回入口时：`站内搜索入口 / 需浏览器核验`。
- 用户提供但自动打不开时：`用户提供 / 需浏览器核验` 或 `自动抓取受限`。
- 如果用户提供浏览器截图或可见结果，再按用户提供信息记录。

如果只返回入口，不要写 `无可核验结果`。需要展示花瓣入口时，只放在主结果表的 `界面/截图` 列或 `待核验线索`。

## 社区和媒体规则

社区、媒体、博客和论坛只作为补充线索。公开搜索限制较大，不能写绝对否定。

可返回的社区/媒体结果必须满足至少一项：

- 页面内有真实 UI 截图。
- 视频或文章明确展示操作路径。
- 能帮助确认产品名称、地区版本、功能入口或官方资源。

纯讨论、新闻、榜单、推广软文且无可见 UI 的结果不要进入主结果；可用于补充竞品池或待核验线索。

## 覆盖不足和否定结论

在声明某平台未找到、无可核验结果、只有安装页或未确认有操作视频前，必须先判断是否存在：

- 目标平台覆盖不足。
- P1/P2 未兜底。
- 名称变体遗漏。
- 旧名/新名、中文名/英文名、包名、开发者名或地区版本遗漏。
- 页面分散、登录限制、地区限制或索引不足。
- 截图过旧、视频不可见或自动抓取受限。

存在这些风险时，必须完成对应兜底后再写覆盖不足。社媒、社区、论坛类平台只能写 `公开搜索未找到可核验结果`，不要写绝对没有。
